## 第一阶段：Bug 修复（必须先做）

### 1.1 单曲循环不工作
**文件**: `static/index.html`
**根因**: `playNext()` (line 1597) 中 "single" 模式走进了 `else { return; }` 分支，不做任何事就退出了。
**修复**:
```javascript
// playNext() 中，将：
if (state.mode === "loop") {
  index = 0;
} else {
  return;
}
// 改为：
if (state.mode === "single") {
  index = state.currentIndex;
} else if (state.mode === "loop") {
  index = 0;
} else {
  return;
}
```
同时 `audio.ended` 事件 (line 2666) 直接调 `playNext()`，需要改成根据 mode 判断：
- single 模式 → 重播当前歌曲 (`audio.currentTime = 0; audio.play()`)
- 其他模式 → 调 `playNext()`

### 1.2 添加歌单偶发失败
**文件**: `static/index.html`, `api.go`
**根因分析**:
1. `api.go:796` — `savePlaylist()` 静默忽略 `json.MarshalIndent` 和 `os.WriteFile` 错误
2. `server.go:245` — POST handler 中 `io.ReadAll(r.Body)` 没有大小限制，但也没有超时保护
3. 前端 `savePlaylist()` 用异步 POST，如果请求失败只 toast 但不重试
**修复**:
- `api.go` savePlaylist: 添加错误日志，失败时返回 error
- `server.go` handlePlaylist POST: 增加 `r.Body = http.MaxBytesReader(w, r.Body, 1<<20)` 限制 1MB
- 前端: 保存失败时重试一次；启动时延迟加载 playlist 确保 server 就绪

### 1.3 B站签名并发安全
**文件**: `signer.go`
**根因**: `sign()` 方法直接修改传入的 `params` map (line 112-117)，并发调用会导致 data race
**修复**: 在 `sign()` 内部拷贝一份 params 再操作

### 1.4 网易云 API 未检查 HTTP 状态码
**文件**: `api.go:286-312`
**根因**: `NeteaseSource.GetURL()` 和 `GetLyric()` 不检查 resp.StatusCode，非200时尝试解析错误JSON
**修复**: 添加 status code 检查

## 第二阶段：UI/UX 重大升级

### 2.1 封面主题色自适应（参考 SPlayer）
**文件**: `static/index.html` (CSS + JS)
**实现**:
- 播放歌曲时，从封面图片提取主色调
- 用 Canvas getImageData 采样封面颜色
- 动态更新 CSS 变量 `--accent`, `--accent2`, `--accent-bg`
- 进度条、按钮、高亮色跟随封面颜色变化
- 切歌时颜色平滑过渡 `transition: all 0.6s`

### 2.2 迷你模式（参考 AlgerMusicPlayer）
**文件**: `static/index.html`
**实现**:
- 新增迷你模式按钮（在标题栏区域）
- 迷你模式：隐藏侧边栏和歌词面板，只保留底部播放栏 + 封面旋转动画
- 窗口缩小到 400x120，点击可恢复完整模式
- 封面做唱片旋转动画（CSS `@keyframes spin`）

### 2.3 播放队列管理
**文件**: `static/index.html`
**实现**:
- 新增"播放队列"侧边面板（从右侧滑入，替代歌词面板切换）
- 显示当前播放队列（搜索结果/歌单/最近播放的完整列表）
- 支持拖拽排序（HTML5 drag & drop）
- 支持从队列中移除歌曲
- 队列为空时显示空状态提示

### 2.4 搜索历史 & 热门推荐
**文件**: `static/index.html`
**实现**:
- 搜索框下方显示最近搜索历史（存 localStorage，最多10条）
- 清空搜索时显示"搜索历史"标签，点击可快速重新搜索
- 每条历史可单独删除

### 2.5 暗色主题（参考 YesPlayMusic）
**文件**: `static/index.html` (CSS)
**实现**:
- 在 CSS 中增加 `[data-theme="dark"]` 选择器的完整暗色变量
- 新增主题切换按钮（在侧边栏底部）
- 主题选择：浅色 / 深色 / 跟随系统
- 主题偏好存 localStorage
- 暗色主题下波形频谱颜色也相应调整

## 验证方案

1. **单曲循环**: 播放一首歌 → 切到单曲循环 → 等歌曲播放完毕 → 验证自动重播
2. **歌单添加**: 搜索歌曲 → 点击添加 → 切到歌单页 → 确认歌曲存在 → 关闭重启 → 再确认歌单已持久化
3. **暗色主题**: 点击主题切换 → 验证所有界面元素颜色正确
4. **封面主题色**: 播放不同封面的歌曲 → 验证 accent 颜色平滑变化
5. **迷你模式**: 点击迷你模式按钮 → 验证窗口缩小、功能可用
6. **播放队列**: 添加多首歌曲到队列 → 拖拽排序 → 删除 → 验证播放顺序
7. **歌词翻译**: 播放网易云歌曲 → 验证翻译歌词显示
8. **右键菜单**: 右键歌曲 → 验证菜单弹出且功能正常
